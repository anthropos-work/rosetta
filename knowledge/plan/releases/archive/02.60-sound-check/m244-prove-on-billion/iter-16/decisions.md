# iter-16 — decisions

## D1 — recalibrate to CARDINALITY, not swap the text token (strengthen the gate, don't weaken it)
The academy home false-FAILED on `mustInclude: ['AI Academy']` because the visible heading is "Academy". The tempting one-liner — swap to `mustInclude: ['Academy']` — would REINTRODUCE the exact gate-(d) false-empty: the pre-fix EMPTY `/library` header read "The AI Academy library" (0 cards), which contains "Academy", so a token check would PASS a genuinely empty catalog. Instead moved to the M42e cardinality shape (`kind: 'both'` + `itemSelector: a[href*="/chapters/"]` + `minCount: 12`): the catalog must actually RENDER course cards. This fails an empty catalog AND a keyless blank (both 0 cards) — strictly stronger than the old token — while keeping the M219 anti-blank floor (`minMeaningfulLen: 400`) and a light wordmark check (`['Academy']`). The unit test was updated to pin the new shape AND to assert the stale "AI Academy" literal is GONE, so the recalibration can't silently revert. Harness-only (coverage-manifest.ts + its unit spec) — the sweep runs from the local authoring copy against billion's unmodified app, so no billion re-pin/re-bake.
