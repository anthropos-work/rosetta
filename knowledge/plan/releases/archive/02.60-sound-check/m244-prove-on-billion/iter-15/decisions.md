# iter-15 — decisions

## D1 — a CHAINED sibling demopatch, not a widened single patch (mirror the codebase's per-patch pattern)
`getServerCatalogView` (authed) and `getPublicCatalogView` (anon) are two distinct lines in the SAME serverTenant.js, differing only by `eids` vs `new Set()`. The M230 patch fixed only the authed line. Options considered: (a) widen the existing manifest to cover both lines (breaks the single-anchor manifest schema), (b) generalize the helper to loop hunks (invasive to a load-bearing helper). Chose (c): a SEPARATE `academy-fs-published-public` manifest + a SEPARATE `apply-academy-fs-published-public.sh` helper — exactly the pattern the codebase already uses for the body half (`academy-fs-published-chapter-body` / `apply-academy-fs-published-body.sh`). Consistency + minimal blast radius over DRY. Because both patches target serverTenant.js, the new one is CHAINED: `pre_sha256` = the sibling's `post_sha256`; applied AFTER FSPUB, reverted BEFORE it (ant-academy.sh enforces the order). This is the documented chain rule (next-web-public-website-url chains on studio's post).

## D2 — `new Set()` (public), not `eids` — the load-bearing tenant-safety difference
The public view MUST filter with the empty/public eid set so no tenant-scoped content can leak onto an anonymous route. The replacement therefore reuses `mergeDrafts(emptyCatalogView(), new Set())` (not `eids`), mirroring the pristine `getPublicCatalogView`'s own `mergeDrafts(view, new Set())`. This keeps the fix public-only by construction.

## D3 — gate (d) is MET on the twin's real-card render; the coverage cross-port's residual is a SEPARATE gate-(c) marker issue
Gate (d)'s criterion is "the anon `/library`+`/free` twin renders real cards." Verified live in a browser: both render (37+/42 cards, 0 Draft chips). So gate (d) is discharged NOW. The coverage sweep's academy-home cross-port still fails, but on a DIFFERENT cause than iter-14 assumed: the home renders 483 real cards, yet the sweep's `mustInclude:['AI Academy']` marker doesn't match the redesigned home's visible "Academy" heading (the literal "AI Academy" is only in the non-visible `<title>`). Chose NOT to fold the marker recalibration into iter-15: it touches the coverage GATE's teeth (`coverage-manifest.ts` + its unit spec) and must preserve the M219 anti-keyless-blank discrimination — a careful, test-backed change that is gate-(c) work, not gate-(d). Routed to iter-16. Refusing to weaken a gate to make it pass is the same discipline as iter-14/D2.
