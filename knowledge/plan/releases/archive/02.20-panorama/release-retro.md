## The release arc

One opt-in capability, delivered in four moves and then proven live:

- **M212 — the single host knob.** ONE opt-in browser-facing knob, `STACK_PUBLIC_HOST` (default `localhost` ⇒ an unset stack is byte-identical to today), surfaced as `/demo-up --public-host <magicdns>` and threaded through every rext emitter that bakes a browser-facing `localhost`/`127.0.0.1`. 12 sections, zero Go touched. The byte-identity contract that made the whole release cheap to review.
- **M213 — auth over the tailnet.** Clerkenstein auth made complete from a **remote** browser and the whole surface served under **one trusted HTTPS MagicDNS origin**: the `mkcert → tailscale cert` path-only drop-in (a genuinely trusted Let's Encrypt cert, no CA install), dotted-pk validation lifted into the demo wiring, the NEW `gen_tailscale_serve.py` reverse-proxy generator (0 net-new deps — `tailscale serve`, already on every VM, chosen over Caddy), and the `cdn.jsdelivr.net` egress made an explicit, overridable seam.
- **M214 — origins & links.** The MagicDNS/HTTPS origin admitted **everywhere a browser→backend or cross-surface call is gated**, driven by **one scheme predicate** (`browser_scheme`/`$SCHEME`: `http` on localhost = byte-identical, `https` for a dotted host): the `CORS_EXTRA_ORIGINS` https trio, the studio-desk redirect + VITE bake, the NEW `apply-ant-academy-dev-origins.sh` demopatch (env-var indirection at a fixed `post_sha256`), and the NEW `corpus/ops/demo/tailscale-serve.md` runbook.
- **M215 — prove it on odyssey.** The iterative acceptance closer and the FINAL milestone: the **FIRST remote Linux-VM demo deploy over Tailscale**, driven end-to-end from a **different tailnet machine** (a remote Mac) for **both** vantages — employee `maya-thriving → /profile` (M41 ProfileSeeder depth rendered) and manager `dan-manager → /enterprise/workforce` (real seeded structural data) — on a **genuinely trusted** LE cert (`ignoreHTTPSErrors:false`, `verify=0`, no CA install), **0 console errors / 0 functional request failures / 0 localhost-or-prod ejects**, reproducibly on a **cold reset-to-seed**. Core exit gate MET; 13 live findings propagated to tools + KB + skills.

## Cross-milestone patterns

- **The byte-identity contract was the spine of the whole release.** Every milestone gated its additions on `--public-host` and pinned them byte-identical when unset — M212's knob (`unset ⇒ exactly today`), M213's dotted-host validator, M214's single `browser_scheme`/`$SCHEME` predicate. The risk surface collapsed to "the opt-in path only," the whole-release review stayed cheap, and it held under the flake gate on every touched suite.
- **One seam per concern, no drift.** M212 one knob, M213 one validator lifted into the wiring, M214 one scheme predicate driving every host-bearing browser surface (CORS, redirects, baked `NEXT_PUBLIC_*`/`VITE_*`, cache-validators, content-URL rewrite, cockpit deep-links). No site could drift out of step with the others.
- **Config-only + a bounded patch tail survived contact with reality (the make-or-break bet, validated at M215).** No new platform edit was invented during the live loop — every breakage routed to the M212 knob / M213 TLS / M214 patch surface or a host-prereq, exactly as the feasibility claim promised. Zero platform-repo edits held end-to-end, and the `tailscale cert` secure-context bet paid off: a **remote** browser got a genuinely trusted cert with **no CA install** — the thing `mkcert` fundamentally cannot do for a second machine.
- **The macOS-only blind spot — the headline structural learning.** The tooling had only ever run on macOS. The first Linux + full-clone + remote run deterministically surfaced a cluster of latent host-assumption bugs (F3 SIGPIPE, F4 ssh-agent, F6 bind-mount perms, F12 serve-reset) that macOS never exposes (Docker Desktop perm-remapping, ssh-agent presence, low tag counts, no persistent tailnet listener). Durable lesson: a "works on my Mac" tooling path needs a real Linux acceptance run before it can be trusted for deploy — and now has one, baked into `test_host_prereqs_m215.py` + the runbook.
- **rext-frozen-per-milestone README reconciliations pile at the code-of-record roll** (recurred from v2.1). Each milestone freezes rext at its annotated tag, so three distinct README index gaps (D-CLOSE-1 demo-stack `test_tooling` count-drift · D-CLOSE-2 stack-injection missing the `gen_tailscale_serve.py` row · D-CLOSE-3 rext READMEs not indexing `apply-ant-academy-dev-origins.sh`) all route to the ONE close-release rext commit. A structural property of the two-repo split, not a lapse.
- **python3.14 runner friction recurred all four milestones (non-blocking).** The box-default `python3` (3.14) has a broken `pyexpat` / no usable pytest; every suite + JUnit tally ran under `python3.12`. Persistent low-grade friction; a standing note for the runner env.
