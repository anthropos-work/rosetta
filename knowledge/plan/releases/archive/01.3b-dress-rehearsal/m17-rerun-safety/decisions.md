| M17-D3 | Replay re-run guard = **TRUNCATE-and-reload, safe-by-default**, NOT an `--idempotent`/`--force` flag. New `Replayer.ClearForReplay` empties every target table (child-first, plain TRUNCATE ‚Äî no CASCADE) BEFORE loading any (parent-first). Resolves overview open-questions 1+2. | A snapshot surface is a public-reference set that fully REPLACES the stack's copy ‚Äî TRUNCATE-then-reload is the simplest correct semantics (overview lean), and the target is always per-stack-isolated (the offset DSN). Safe-by-default (no flag) because the operation is harmless on a first run (TRUNCATE of empty = no-op) and on a re-run is the intended behavior. | 2026-06-08 |
| M17-D4 | The TRUNCATE SQL is built by a single pure func `truncateForReplaySQL` (adapters.go), pinned by a **target-class test** to ALWAYS be a `TRUNCATE TABLE "schema"."table" RESTART IDENTITY` ‚Äî never DROP/DELETE/CASCADE/cross-schema, identifiers QuoteIdent-escaped (injection-safe). | The load-bearing prod-safety constraint (safety.md): a wrong-target TRUNCATE = data loss. | M17-D5 | Seed deterministic-id idempotency = a NEW `Conn.CopyRowsIdempotent(... conflictCol)` that COPYs into a session-local `TEMP TABLE (LIKE ‚Ä¶ INCLUDING DEFAULTS) ON COMMIT DROP` then `INSERT ‚Ä¶ SELECT ‚Ä¶ ON CONFLICT (id) DO NOTHING` into the real table, in ONE tx. Applied uniformly (not just the 2 surfaces the overview named) ‚Äî they were examples of the whole deterministic-id class (three-fate Fate-1: full work). | 2026-06-08 |
