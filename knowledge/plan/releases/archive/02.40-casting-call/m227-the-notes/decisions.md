| **D2** | **Fix #2 = single-source `emailForMember(st, role, first, last, seed, idx)`, keyed on role.** Both `users.go` (seeded email) + `roster.go` (Clerkenstein login identity) call it with the same uid/idx/role → byte-identical. | **D3** | **Fix #3 = round-robin each assessed candidate onto ONE position by ordinal; floor = 6.** Measured 43 assessed → min 8 / max 9 per position (round-robin, max-min<=1). Round-robin gives the tightest, most predictable split (vs hash-mod). | 2026-07-17 |
| **D4** | **Fix #4 = curated first-name→gender dictionary + gender-partitioned face pool; Unknown → byte-identical full-pool fallback.** No library, no LLM re-generation (works on the cached seed). Threaded at all 3 avatar call sites with the same (seed, first) so menu==profile survives. | Deterministic, offline, $0. Unisex/unknown names honestly fall back (never a forced wrong-gender face) and reproduce the old pick exactly. | 2026-07-17 |
