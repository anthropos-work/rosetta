# iter-02 — Decisions

_Implementation choices for `GUARD-M221-host-isolation`, with rationale._

| # | Decision | Rationale |
|---|----------|-----------|
| D-M221-02a | **A host lock (not per-cycle N).** A per-N lockfile taken for the life of a cycle, over the overview's alternative of "a per-cycle stack N." | The lock directly names the holder in the refusal (the DoD's headline) and — crucially — protects the **teardown** path, which is where M219 actually broke (a teardown purged the stack mid-measurement). Per-cycle-N would isolate *allocation* (the unified registry already does that), but it does **not** stop two agents from driving the *same* N, which is exactly what happened. Lighter than standing up N-isolation machinery. |
| D-M221-02b | **Atomic take via `set -C` (O_EXCL), never check-then-write.** `( set -C; printf … > "$f" )` — a real exclusive create. | The whole safety property is that two simultaneous acquirers can never both win. `[ -e "$f" ] && …` is a TOCTOU race — the very "no mutual exclusion on a shared host" bug this guard exists to kill. Proven both statically (source assertion) and dynamically (24 racers → exactly 1 winner; the check-then-write mutant races). |
| D-M221-02c | **Lock lives at `~/.rosetta-demo-locks/`, OUTSIDE any `$STACK`.** | `down --purge` wipes `$STACK/data`, and a re-clone rewrites the ephemeral clone; a lock under either would be clobbered by the very cycle it must outlive. `$HOME` survives both. |
| D-M221-02d | **Reclaim = age > TTL AND holder provably gone; and it is LOUD.** Same-host liveness by `kill -0`; a cross-host holder is treated as "not gone" (unprovable). TTL default 7200 s (a full cycle is ~15–45 min → generous). | "The file exists" ≠ "a live cycle holds it" (D17). A crashed cycle (`kill -9`, untrappable) must not deadlock the host forever — so a *provably dead* + *stale* lock is reclaimable. But a reclaim is a takeover of someone's advertised hold, so it must be announced (logs the dead holder), never silent. A live holder is never reclaimed, even at `TTL=0`. The demo-host topology is co-located (the cycle and its lock are on `billion`), so `kill -0` is decidable in the real case. |
| D-M221-02e | **One consolidated `_up_injected_atexit` handler in `up-injected.sh`; `hostlock_acquire` runs `HOSTLOCK_NO_TRAP=1` there.** The old bare `trap reap_ssh_agent EXIT` inside `ensure_ssh_agent` is removed and folded in. | bash keeps exactly ONE EXIT trap. `ensure_ssh_agent` sets its EXIT trap *mid-build on billion* — it would have CLOBBERED a hostlock EXIT release set earlier (orphaning the lock until stale-reclaim). Consolidating into one handler set once, with the acquire told not to install its own competing traps, makes the release un-clobberable. In `rosetta-demo` (no competing EXIT trap) `hostlock` installs its own traps normally. |
| D-M221-02f | **A `HOSTLOCK_LIB`-style mutation seam in the test file.** The same behavioural fences run against four targeted mutants. | The release's core discipline: a fence that passes without the guard is theatre (it happened repeatedly this release). Mutation-proving makes each fence self-certify as load-bearing, in addition to the whole-suite RED-proof against the pre-fix tree (18 failed → 18 passed). |
