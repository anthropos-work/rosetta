# iter-03 — Decisions

_Implementation choices for the Phase B off-box demo-hygiene cluster, with rationale._

| # | Decision | Rationale |
|---|----------|-----------|
| D-M221-03a | **Fix (1) passes an EXPLICIT `-H 127.0.0.1` on the localhost path, not "omit the flag".** `bind_args=(-H 127.0.0.1)` is the un-gated default; `-H 0.0.0.0` stays gated on `STACK_PUBLIC_HOST`. | The old code trusted "next dev's own default" — but that default IS `0.0.0.0`, which is the entire F-M220-5 bug. "Byte-identical to today" was the bug talking: today the academy world-publishes. The explicit loopback is a strict de-exposure. |
| D-M221-03b | **The M220 S3 HARD INVARIANT is NOT disturbed** — confirmed by keeping `TestTheFallbackIsByteIdentical` green. | That invariant is fenced on `up-injected.sh`'s derivation trio (`BIND_HOST`/`HOST`/`SCHEME` = `""`/`localhost`/`http`) + zero tailscale probe — none of which fix (1) touches (`BIND_HOST` stays `""` on localhost). The academy's `-H` is a separate lever the invariant never covered; tightening it mints no cert, probes no tailscale, bakes no different URL. |
| D-M221-03c | **The exposure guard reads the TOOL DEFAULT, not just the launch flag.** `NEXT_DEV_DEFAULT_BIND=0.0.0.0` (named + commented); `cockpit.py`'s `--host` default is parsed from its argparse source. | The bug hid *because* the script wrote no flag and the effective bind was the tool's default. A guard that only checked "does the script pass `-H`?" would answer "loopback" for the very code that world-published. Same construct, opposite tool defaults (next→0.0.0.0, cockpit→127.0.0.1) — so the tool default must be read, from source, never a hand-copied constant. |
| D-M221-03d | **An unparseable host-native bind construct returns 2 (a FINDING), never a silent pass.** | The exposure fence's own doctrine (D17): an empty/unreadable exposure answer is a finding. If a future rewrite of the bind construct breaks the parser, the guard fails loud and forces a deliberate re-pin — it does not shrug and pass. |
| D-M221-03e | **Fix (2) is delivered as a FENCE; the reap CODE already shipped (M217 `reap_native_ports` + M220 S5(i) pre-bind reap) but was UNFENCED.** The launcher test harness never even copied `reap.sh` into its section dir, so `[ -f reap.sh ]` was always false and the reap block ran in ZERO tests. | An unfenced fix is exactly the D17 shape that regresses silently. The genuine gap was coverage of the `ant-academy.sh` pre-bind wiring — landed as an integration fence (copy `reap.sh`, pre-hold the port with an academy-identity listener, prove reclaim + the new bind), mutation-proven by excising the reap block (stale academy wins, launcher reports not-serving). |
| D-M221-03f | **Fix (2)'s cross-N safety is proven at the PORT scope** (a listener on a different offset port survives `ant-academy.sh N`). | `reap_port` examines only listeners ON `$PORT`, so a co-resident demo-M academy (different offset port) is never even inspected. Port-scoping is structural; the test pins it. reap_port's identity/foreign safety is separately mutation-proven in `test_reap.py` — not re-litigated here. |
| D-M221-03g | **Fix (3)'s DoD is the FENCE, not a live re-measure** (the live blackhole belongs to Phase C on `billion`). The guard flags a reader in a **definitively server-side execution context**, not "any non-`'use client'` module." | Flagging every non-`'use client'` module would RED the shipped tree (context-neutral shared libs read the var and are reached only from client code — M218 D10) — a false alarm, not a fence. The actionable, non-noisy, GREEN-today interpretation of "server-side reader" is: route handler, app-router server component (app/ special file w/o `'use client'`), middleware, `getServerSideProps`/`getStaticProps`/`getInitialProps`, `server-only` import. Same scope as the M218 SSR fence: textual readers, not transitive server callers of a client-lib fetch (stated honestly in the guard). |
| D-M221-03h | **Fix (3) enforces M218 D10 rather than restating it** — "all readers are client-side today" becomes a live-clone gate + a real-source mutation proof, plus a zero-readers FINDING guard. | The release's signature hazard (D17) is a status claim read as evidence after the thing changed. "All readers are client-side" is precisely such a claim; it needed a fence, not a comment. The real-clone gate skips-not-passes when the clone is absent (a coverage hole ≠ a pass, M219). |
