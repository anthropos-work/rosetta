# iter-05 — Decisions

_Fix + strategy choices made while landing F1/F2/F5 and re-proving, with rationale._

| # | Decision | Rationale |
|---|----------|-----------|
| D-M221-05a | **F1 is fixed in THREE places, not one.** (A) `workspaceRootFrom` now PREFERS an ancestor whose `.agentspace` holds `snapshots/` over a nearer empty marker; (B) `dev-setdress.sh` resolves the workspace cache root and passes an explicit `--store` on every replay; (C) `replay` is LOUD about an EMPTY store (the wrong-root smell). | iter-04 diagnosed the seam as `dev-setdress.sh:323` (no `--store`), but the ROOT is stacksnap's cwd walk-up preferring the nearest `.agentspace`. Fixing only the caller (B) leaves every OTHER caller exposed; fixing only the resolver (A) leaves the resolution cwd-dependent. Both + the D17 loud fence (C) make the resolution correct, deterministic, AND self-diagnosing. Each RED-fenced independently. |
| D-M221-05b | **The D17 "fail loud" is a LOUD non-fatal diagnostic on an EMPTY store — NOT a new fatal exit.** | The directive is "fail loud, not silently load 0." Making an empty store FATAL would break the M20 atomicity contract (a genuinely cold box with no capture yet must degrade gracefully to structural-only, not abort). So the empty-store path stays exit 5 (non-fatal skip) but now SAYS "the store is EMPTY — wrong root? the real cache may sit in a parent .agentspace". Loudness is the fence; fatality would regress graceful degradation. |
| D-M221-05c | **F5's real mechanism is a SUPERVISED RESPAWN, not a pidfile gap.** The reap now kills identity-matching ANCESTORS (the `next dev` supervisor), not just the socket-holding worker. | The teardown ALREADY reaped by port+identity (`reap_native_ports`), yet the academy survived on :13077 and teardown reported "free". A dev server supervises + RESPAWNS its worker: killing only the listener let `next dev` re-bind, and the re-probe raced past the respawn. Killing the identity-matching parent chain (stops at the first non-matching ancestor — never a blind pgroup kill, never crosses pid 1) removes the respawn source. RED-fenced with a real supervisor→worker respawn harness. |
| D-M221-05d | **F5b is a `--stop` path-scoped `git checkout`, NOT a gitignore.** | Gitignoring the two predev-hook-generated files (`code/public/catalog.json` + `content/index.md`) would edit the ant-academy repo's `.gitignore` — a forbidden repo edit. Restoring them to their committed state on `--stop` (a tooling action, idempotent no-op when pristine) leaves the clone git-clean for the gate-8 check without touching any tracked file's content. |
| D-M221-05e | **F2 (directus re-capture) is NOT pre-emptively re-captured — the F1 fix likely unblocks directus via M21 auto-provision, verified live.** | iter-04's F2 divergence (`b4cb55bc` wanted vs `ea2e187a` cached) was measured against the BROKEN store — the auto-provision found an empty store and could not apply the cached Structure. The directus cache HAS a `_structure.sql` artifact (confirmed on billion), so with F1 fixed the M21 auto-provision should bridge the bootstrapped-gap schema to the captured one. Re-capturing directus needs a privileged prod source; doing it speculatively before confirming F1 doesn't already fix it would be wasted (and risk). Measure live; re-capture only if it still misses. |
| D-M221-05f | **The re-prove FALSIFIED F1's fix on the live box.** The shipped 3-place fix (A resolver-prefers-snapshots, B `dev-setdress.sh --store`, C loud-empty-store) did **NOT** achieve `skills>0` on a real cold `up-injected.sh 1` on billion: the cold cycle came up **`green:false, warnings:3`**, **`public.skills=0`** (unchanged from the iter-04 baseline), and **directus exited(1)** on a postgres `get_namespace_oid` schema error (the empty-taxonomy DB never got the schema directus expects). iter-04 "de-risked to certainty" by MANUALLY pinning `STACKSNAP_STORE`, which loaded 42,790 rows — but the shipped automated resolution did **not** reproduce that manual pin against billion's real directory topology. This is the release's signature shape once more: a fix proven against the agent's model / unit fences that does not survive the live box. **The orchestrator graded this cycle** (the iter-05 agent ran out of budget after landing the fixes but before grading — D-M221-05f was left blank). **Routed to iter-06:** re-diagnose WHY the store resolution still lands on an empty root on billion (fixes A+B did not take), and whether the directus schema failure is a consequence of the empty taxonomy or an independent ordering bug. |
