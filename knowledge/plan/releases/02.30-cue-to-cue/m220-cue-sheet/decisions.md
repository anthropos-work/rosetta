# M220 — Decisions

_Implementation choices with rationale. One entry per decision; cite the code/doc it binds._

| # | Decision | Rationale | Date |
|---|----------|-----------|------|
| **D1** | **The fences DERIVE the truth from code; they never restate it.** `story_org_count_guard` **counts** `org:` entries in the preset (a 4th story ⇒ every stale prose site goes RED). `exposure_claim_guard` **runs** the three port emitters. `demo_knob_guard` **parses** the two arg-parsers. | A fence that hardcodes `assert count == 3` rots exactly like the prose it replaced. The `seed_label` in `up-injected.sh` was a hardcoded "2 orgs" that outlived the preset by four releases — restating a remembered number *is* the defect (D17). It now `grep -c`s the preset. | 2026-07-14 |
| **D2** | **A retracted claim is quoted inside a FENCED BLOCK; live prose is scanned.** `exposure_claim_guard` skips fenced blocks (incl. blockquote-nested). | A doc that corrects a false safety claim must be able to **quote** it. Without this the guard is un-satisfiable in the one document it exists to fix — you could never write the correction without tripping it. Prose *asserts*; a fenced block *displays*. | 2026-07-14 |
| **D3** | **Doc scans operate on FLATTENED text (sentence-bounded), not physical lines.** | Markdown prose **wraps**. The shipped lie was split across a line break (`"A bare \`/demo-up N\`"` \| `"binds loopback only…"`), and a per-line scanner reported a confident 1-hit result **while a second live lie sat one line below, inside the block it had just audited**. | 2026-07-14 |
| **D4** | **Exclusions match path COMPONENTS, never absolute-path substrings.** | `.agentspace` was in `story_org_count_guard`'s exclusion list, matched as a substring. rext lives at `<rosetta>/.agentspace/rosetta-extensions/` — so the guard **silently skipped both of its own repo's sites** while printing a confident failure list for the other five. An exclusion that matches the absolute path can swallow the whole repo. | 2026-07-14 |
| **D5** | **Truth is PER-PRESET, and measurements are not claims.** `stories-maya.seed.yaml` genuinely ships **1** org; run-reports (*"1 org on the M215 run"*) are correct measurements of a cold-cache run; *"each entry is ONE org"* is schema cardinality. All three are suppressed. | A single global truth flagged three TRUE statements as lies. Rewriting the M215 measurement to "3" would have replaced a correct observation with a fabrication — **the guard would have caused the defect class it exists to catch.** | 2026-07-14 |
| **D6** | **`safety.md` Part 3 is a DISCLOSURE, not a third "never".** | Parts 1–2 are structural guarantees (a prod read/write is made *impossible*). Part 3 cannot be — **a demo nobody can reach is a demo that does not work.** So it states what is exposed, to whom, and what a reacher **obtains**, and rests the argument on that last clause. Pretending otherwise would be a fourth lie in a doc written to end lies. | 2026-07-14 |
| **D7** | **The supersession is written where the superseded decision LIVES** (`demo-up/SKILL.md`), and always cited as **"v2.2's D-DESIGN-1"** — never bare. | v2.3 has its **own** `D-DESIGN-1` (the <5 s ACCESS gate). A bare citation resolves to the wrong decision. Never a silent contradiction. | 2026-07-14 |
| **D8** | **`demo-up/SKILL.md` is NOT changed to say "default-on".** It still documents the flag as required, + a forward-pointer that the flip is **S3**. | The code still requires the flag. A doc claiming a behavior the code lacks **is the exact defect S0 just fixed.** The supersession is a *decision*; the flip is *code*. Documenting the decision ≠ pre-announcing the code. | 2026-07-14 |
| **D9** | **The conflation rule.** A flag in the skill's `argument-hint` must be accepted by the **primary** entry point (`up-injected.sh`), unless the hint **names** the other script. | `--profile`/`--services` **are** parsed — by `rosetta-demo`. So a fence asking *"does SOME parser take it?"* goes **green** while `up-injected.sh N --profile cms` still **exits 1**. A check that proves the wrong proposition is the bug. RED-proven against the live hint (2 conflations). | 2026-07-14 |
