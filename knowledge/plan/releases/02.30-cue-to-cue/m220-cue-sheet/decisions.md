# M220 — Decisions

_Implementation choices with rationale. One entry per decision; cite the code/doc it binds._

| # | Decision | Rationale | Date |
|---|----------|-----------|------|
| **D1** | **The fences DERIVE the truth from code; they never restate it.** `story_org_count_guard` **counts** `org:` entries in the preset (a 4th story ⇒ every stale prose site goes RED). `exposure_claim_guard` **runs** the three port emitters. `demo_knob_guard` **parses** the two arg-parsers. | A fence that hardcodes `assert count == 3` rots exactly like the prose it replaced. The `seed_label` in `up-injected.sh` was a hardcoded "2 orgs" that outlived the preset by four releases — restating a remembered number *is* the defect (D17). It now `grep -c`s the preset. | 2026-07-14 |
| **D2** | **A retracted claim is quoted inside a FENCED BLOCK; live prose is scanned.** `exposure_claim_guard` skips fenced blocks (incl. blockquote-nested). | A doc that corrects a false safety claim must be able to **quote** it. Without this the guard is un-satisfiable in the one document it exists to fix — you could never write the correction without tripping it. Prose *asserts*; a fenced block *displays*. | 2026-07-14 |
| **D3** | **Doc scans operate on FLATTENED text (sentence-bounded), not physical lines.** | Markdown prose **wraps**. The shipped lie was split across a line break (`"A bare \`/demo-up N\`"` \| `"binds loopback only…"`), and a per-line scanner reported a confident 1-hit result **while a second live lie sat one line below, inside the block it had just audited**. | 2026-07-14 |
| **D4** | **Exclusions match path COMPONENTS, never absolute-path substrings.** | `.agentspace` was in `story_org_count_guard`'s exclusion list, matched as a substring. rext lives at `<rosetta>/.agentspace/rosetta-extensions/` — so the guard **silently skipped both of its own repo's sites** while printing a confident failure list for the other five. An exclusion that matches the absolute path can swallow the whole repo. | 2026-07-14 |
| **D5** | **Truth is PER-PRESET, and measurements are not claims.** `stories-maya.seed.yaml` genuinely ships **1** org; run-reports (*"1 org on the M215 run"*) are correct measurements of a cold-cache run; *"each entry is ONE org"* is schema cardinality. All three are suppressed. | A single global truth flagged three TRUE statements as lies. Rewriting the M215 measurement to "3" would have replaced a correct observation with a fabrication — **the guard would have caused the defect class it exists to catch.** | 2026-07-14 |
| **D6** | **`safety.md` Part 3 is a DISCLOSURE, not a third "never".** | Parts 1–2 are structural guarantees (a prod read/write is made *impossible*). Part 3 cannot be — **a demo nobody can reach is a demo that does not work.** So it states what is exposed, to whom, and what a reacher **obtains**, and rests the argument on that last clause. Pretending otherwise would be a fourth lie in a doc written to end lies. | 2026-07-14 |
| **D7** | **The supersession is written where the superseded decision LIVES** (`demo-up/SKILL.md`), and always cited as **"v2.2's D-DESIGN-1"** — never bare. | v2.3 has its **own** `D-DESIGN-1` (the <5 s ACCESS gate). A bare citation resolves to the wrong decision. Never a silent contradiction. | 2026-07-14 |
| **D8** | **`demo-up/SKILL.md` is NOT changed to say "default-on".** It still documents the flag as required, + a forward-pointer that the flip is **S3**. | The code still requires the flag. A doc claiming a behavior the code lacks **is the exact defect S0 just fixed.** The supersession is a *decision*; the flip is *code*. Documenting the decision ≠ pre-announcing the code. | 2026-07-14 |
| **D9** | **The conflation rule.** A flag in the skill's `argument-hint` must be accepted by the **primary** entry point (`up-injected.sh`), unless the hint **names** the other script. | `--profile`/`--services` **are** parsed — by `rosetta-demo`. So a fence asking *"does SOME parser take it?"* goes **green** while `up-injected.sh N --profile cms` still **exits 1**. A check that proves the wrong proposition is the bug. RED-proven against the live hint (2 conflations). | 2026-07-14 |

| **D10** | **The academy is CLERKENSTEIN-WIRED, and the `e2e_persona` bypass is DELETED — not kept "just in case".** | The bypass existed only to fake an authenticated session on a **keyless** academy — a workaround for the wiring gap S5 closes. Kept *alongside* real keys it is **worse than either alone**: `proxy.js` short-circuits on the persona cookie **before** it resolves the real session, so the academy would render a generic **"E2E Member"** to a presenter logged in as **Maya** — a persona self-consistency defect shipped by our own launcher. A fallback that produces a *believably wrong* demo is not a fallback. | 2026-07-14 |
| **D11** | **The fake BAPI is published on `127.0.0.1` — the demo's FIRST loopback-bound port.** | ant-academy is the demo's one **host-native** frontend (`next dev`, never dockerized), so it cannot resolve the compose alias `api.clerk.com` that every container uses. Without a published port its only reachable `CLERK_API_URL` is the **default: real `api.clerk.com`** — the demo phoning Clerk, which is exactly what S6 exists to stop. It is bound to **loopback** precisely *because* S0/S1 established every other port is world-published on `0.0.0.0`: a **disarmed** mock that ignores the bearer entirely is the last thing that should be ambient on a tailnet. Narrowest bind that works. | 2026-07-14 |
| **D12** | **`next-web-no-thirdparty` gates the scripts behind an env var rather than deleting them.** | Same rule as `next-web-studio-url`: the replacement keeps every tag **byte-for-byte** inside a build-time gate, so with the var unset the behaviour is **identical to pristine** — the patch is safe even if it were ever upstreamed, and the diff reads as a *seam*, not a *deletion*. A patch that deletes code is a patch the next platform bump will fight. | 2026-07-14 |
| **D13** | **The image-reuse check now keys on a PATCH-SET FINGERPRINT (an image label), not just endpoint + pk.** | **Applying a patch is not shipping it.** Reuse was keyed on two things with **zero relationship to the patch set**, so an image built before a patch was added passed both checks and was reused — the patch applied to the clone, reverted afterwards, and **never reached the image**. Green bring-up, unpatched bundle, nothing said so. That is the mechanism behind demopatch-spec's own war story (a perf patch that shipped a 76 s members grid for **four releases**). A **label** carries the fingerprint with **no Dockerfile edit**, so the zero-platform-edit line holds. Fail-safe: an unlabelled image predates the fingerprint ⇒ rebuild. **It fired on its first live run.** | 2026-07-14 |
| **D14** | **The clerk-js cache is BOX-level, and the cache dir is created by `up-injected.sh` — never by `gen_injected_override.py`.** | Box-level (not per-demo) so the **second** demo on a box never touches the internet for clerk-js at all; keyed by the request path's `package@version`, which is **self-invalidating by construction** (a stale entry is impossible). The mkdir lives in the shell because `build_lines` is a **pure builder** that the S0 exposure fence *imports and runs with a synthetic data_root* — a filesystem side effect at emit time crashed the safety fence with `Read-only file system: '/.clerkjs-cache'`. **A builder that touches the disk is not a builder.** | 2026-07-14 |
| **D15** | **S5(j) was NOT a session bug, and the milestone plan's diagnosis was wrong.** | The plan asserted studio-desk "doesn't recognise the Clerkenstein session". Measured: **it does.** The roster already seeds each hero's `(org, role)` into the fake BAPI, and `dan-manager` (role `admin`) passes `checkEnterpriseAndAdmin` and **stays in Studio**. M219's evidence — *"`:19000` → 302 → `/login`"* — was a **cookieless `curl`**, i.e. the *correct* answer to an unauthenticated request, mis-read as the defect. An employee is still redirected off Studio, which is the **real platform's** behaviour and must not be "fixed". **The check proved the wrong proposition — that was the bug, and it was in the plan, not the code.** | 2026-07-14 |
| **D16** | **Session survival and academy CONTENT are asserted as SEPARATE fences.** | They are different claims and must not mask each other. On the first run the 400-char content floor tripped **before** the session assertion ran — so the DoD went **unmeasured** and the result merely *looked* like a session failure. An unexecuted check is a FINDING, not a pass; an unexecuted check **reported as a failure of something else** is worse. The content floor now lives in its own test, where RED means *"the academy is empty"*, never *"the session died"*. | 2026-07-14 |

| **D17** | **The ladder is CAPABILITY-gated, and rung 6 demands a CERTIFICATE — not a binary.** Six rungs; a failed rung is a *failure*, never a downgrade. | Rungs 1–5 all pass on a box where the mint still fails (no operator, tailnet HTTPS off, an ACME hiccup) — and `gen_tailscale_fapi_cert` then **silently falls back to a local-trust cert that the REMOTE browser rejects**. That is a green bring-up nobody can use, on the one machine the feature exists for. *"The binary exists"* is not *"it works."* The dotless-name rung is likewise a **hard refusal**, not a downgrade: the host is baked into the pk, and `assertValidPublishableKey` **500s every request** on a dotless one. | 2026-07-14 |
| **D18** | **Any failed rung ⇒ an EMPTY host. Never a partial one.** The `\|\| true` in `resolve_public_host` and the module's always-0 exit are **load-bearing**, not defensive habit. | `SCHEME` and `BIND_HOST` derive from the **same `-n $STACK_PUBLIC_HOST` predicate**, so a *half-satisfied* public path is **strictly worse than localhost**: every baked URL flips to `https://` against plain-HTTP listeners and the demo **does not load at all**. A localhost demo always works. Fenced with a **crashing** discoverer: drop the `\|\| true` and a tailscale hiccup **aborts the bring-up**, leaving a box that can no longer run *any* demo — not even the localhost one it ran yesterday. | 2026-07-14 |
| **D19** | **The cockpit is its OWN axis in `gen_tailscale_serve.py` — not a member of `UI_BROWSER_FACING`.** (The plan said *"add `('cockpit', 7700)` to `:42-46`"*, i.e. the UI list; this deviates deliberately.) | The knob that governs the **listener** must be the knob that governs the **proxy**. The cockpit is gated on `DEMO_STORIES`, **not** on `--no-ui` (`up-injected.sh:1563`). Filing it under the UI tier would have left a **live** cockpit **unfronted** on a `--no-ui` stories demo — a plain-HTTP hero-launcher on the tailnet, the exact hole S4 exists to close — and fronted a **dead port** under `DEMO_NO_STORIES=1`. | 2026-07-14 |
| **D20** | **Bind first, front second.** The first serve apply passes `--no-cockpit`; a **second**, idempotent apply fronts `:7700` only after the cockpit answers `/healthz`. The **reset** plan always includes `:7700`. | `tailscale serve` binds the tailnet IP `:<port>` as a **REAL listener** (M215 F12). Fronting `:7700` at the original apply site — ~250 lines **before** the cockpit launches — makes the cockpit's own bind fail `EADDRINUSE`: we would have "fixed" the cockpit's exposure **by killing the cockpit**. Already observed live for `ant-academy` (F-M220-4). The reset asymmetry is the same coin: skip `:7700` there and a re-up over a stale serve config finds the port held and the cockpit **cannot start at all**. | 2026-07-14 |
| **D21** | **S4 is TRANSPORT, not AUTHENTICATION — and the docs say so in those words.** | Fronting the cockpit on `tailscale serve` puts it behind the tailnet's TLS + authenticated device mesh instead of cleartext HTTP. It does **not** password-protect it: it remains a one-click, password-free *"become any seeded hero"* launcher (a bare `GET /v1/client/handshake?__clerk_identity=<key>`). A safety doc that let a reader infer otherwise would be a **new** lie in a milestone written to end them. What makes the trade acceptable is structural and unchanged: **no customer data, no prod writes** (safety.md Parts 1–2). | 2026-07-14 |
| **D22** | **The `tailscale cert` re-issue claim was settled EMPIRICALLY before the flip shipped, not trusted.** | rext asserted in two places that `tailscale cert` **RE-ISSUES on re-run**. If true, default-on — whose rung 6 **mints on every bring-up** — would burn a Let's Encrypt **duplicate-certificate** slot per `demo-up`, and **`ts.net` is a PSL entry**, so that bucket is **per-tailnet**: one busy box could rate-limit everyone. **Measured: FALSE.** Two back-to-back mints → **identical serial**, **0.01 s**, **zero** ACME orders. The flip was gated on the measurement; the docs were the bug. | 2026-07-14 |
| **D23** | **The academy's loopback-bind fix is NOT bundled into S3/S4 — it is routed to M221.** | S3's whole fenced invariant is that the **localhost path stays byte-identical**. Passing `-H 127.0.0.1` to `next dev` would *change* that path (for the better) — and it is exactly the kind of unproven behaviour change that must not ride along inside the milestone whose thesis is *"the docs must match the code"*. So M220 **corrects the doc** (Fate 1 — a false safety claim ships nothing) and **routes the code** (Fate 3). | 2026-07-14 |
